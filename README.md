# Galaxy Classifier Web Application

An end-to-end web application for browsing galaxy images and generating predictions using a deep learning model deployed on AWS SageMaker.

This project combines a modern web interface with cloud-based machine learning inference to classify galaxy images.

Users can:

  * Browse randomly selected galaxy images
  * Shuffle to load new images
  * Upload their own images
  * Generate predictions using a trained deep learning model
  * Visualize model predictions and confidence scores

The application demonstrates how a machine learning model can be integrated into a production-style web architecture using AWS services.

---

# System Architecture

Browser
   ↓
Angular Frontend
   ↓
ASP.NET Core API
   ↓
Amazon S3 (image storage)
   ↓
Amazon SageMaker Endpoint (ML inference)

The API generates **pre-signed URLs** for private S3 images and calls the SageMaker endpoint for predictions.

---

# Responsibilities

Angular Frontend

  * Displays image gallery
  * Handles image uploads
  * Requests predictions
  * Renders prediction results

ASP.NET Core API

  * Generates pre-signed S3 URLs for private images
  * Validates and Uploads images to S3
  * Sends inference requests to SageMaker
  * Returns prediction results to the frontend

Amazon S3

  * Stores galaxy images
  * Stores uploaded images
  * Stores image manifest used for fast loading

Amazon SageMaker

  * Hosts the trained galaxy classification model
  * Returns prediction probabilities for each image

# Application Features

### Galaxy Image Gallery

Features include:
* Displays a variety of images loaded from S3
* Shuffle button loads new images
* Upon prediction overlays the predicted image with a heatmap that shows the parts of the image that contributed most to the model's prediction

Datasets Include:
* Galaxy Zoo images
* Negative astronomy samples
* STL10 negative samples

### Image Upload

Users can upload their own image for prediction.

Workflow:
* User selects an image from their device
* Image is uploaded to S3 through the API
* A prediction request is sent to SageMaker
* Results are returned to the frontend

Uploaded images are stored under a dedicated S3 prefix.

### ML Predictions

Predictions are generated by two deep learning models (resnet18 and custom CNN), trained on Galaxy Zoo data.

The prediction response includes:
* predicted label for each model
* prediction confidence for each model
* probability distribution across classes

The frontend displays prediction cards showing the results.

# Local Development Setup

## 1. Clone the repository

```
git clone <repo-url>
cd galaxy-app
```

---

## 2. Configure AWS credentials

Local development uses **.NET user secrets** so credentials are not stored in the repository.

Initialize secrets:

```
cd api/GalaxyClassifier
dotnet user-secrets init
```

Add credentials:

```
dotnet user-secrets set "AWS_ACCESS_KEY_ID" "your-access-key"
dotnet user-secrets set "AWS_SECRET_ACCESS_KEY" "your-secret-key"
dotnet user-secrets set "AWS_REGION" "us-east-1"
```

Verify:

```
dotnet user-secrets list
```

---

## 3. Run the backend API

```
cd api/GalaxyClassifier
dotnet run
```

The API will start at:

```
http://localhost:5165
```

Swagger UI:

```
http://localhost:5165/swagger
```

---

## 4. Run the Angular frontend

```
cd client
npm install
ng serve
```

The frontend will start at:

```
http://localhost:4200
```

The Angular dev server proxies `/api` requests to the backend.

# Configuration

### appsettings.json

```
{
  "GalaxyS3": {
    "BucketName": "your-s3-bucket",
    "ImagePrefixes": [
      "galaxy-classifier/data/images/galaxy_zoo/",
      "galaxy-classifier/data/images/astro_neg/",
      "galaxy-classifier/data/images/neg_stl10/"
    ],
    "ManifestKey": "galaxy-classifier/manifests/image-manifest.json"
  },
  "SageMaker": {
    "EndpointName": "galaxy-classifier-endpoint"
  }
}
```

---

# Image Manifest

To avoid expensive S3 listing operations, the API loads an **image manifest** containing all image keys.

Example:

```
{
  "galaxy-classifier/data/images/galaxy_zoo/": [
    "galaxy-classifier/data/images/galaxy_zoo/img1.jpg",
    "galaxy-classifier/data/images/galaxy_zoo/img2.jpg"
  ]
}
```

The manifest is stored in S3 and loaded by the API.

# Deployment

The easiest deployment path is to deploy the **combined frontend and backend** to Elastic Beanstalk.

### Steps

1. Build Angular

```
cd client
ng build
```

2. Copy build output into the API `wwwroot` folder

3. Publish the .NET app

```
dotnet publish -c Release
```

4. Deploy the published output to an **Elastic Beanstalk environment**

Production authentication should use an **EC2 instance IAM role** instead of static AWS credentials.
